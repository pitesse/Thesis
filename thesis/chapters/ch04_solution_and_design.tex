\subsection{Implemented Pit-Cycle Fallback Hierarchy}

For \texttt{PitStopEvaluator}, the implemented design is a strict resolution hierarchy:

\begin{enumerate}
	\item direct rival-gap evaluation after both cars complete settle laps,
	\item bounded near-lap gap recovery around the comparison lap,
	\item pace-shift fallback when post-gap remains unavailable,
	\item explicit unresolved emission with reason code if evidence is still insufficient.
\end{enumerate}

The pace-shift fallback is computed as a relative pre/post pace change against the same rival:

$$
\Delta_{\text{pace}\%} =
\frac{(\bar{t}^{post}_{d} - \bar{t}^{post}_{r}) - (\bar{t}^{pre}_{d} - \bar{t}^{pre}_{r})}{t_{baseline}} \cdot 100,
$$

where $\bar{t}$ are local green, non-pit moving averages and $t_{baseline}$ is the pre-pit baseline lap time with conservative fallback. This keeps the decision grounded in lap-time component reasoning and race-time minimization logic \citep{heilmeier2020race, carrasco2023optimization}, while remaining compatible with online, partial-information updates \citep{quiroga2024optimization}. Incident-like extremes are still filtered into unresolved labels, so the fallback increases yield without relaxing label-purity guardrails.

Suggested references: \citep{carrasco2023optimization, quiroga2024optimization}.
